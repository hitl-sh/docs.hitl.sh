## API Authentication Issues

### 401 Unauthorized Errors

<AccordionGroup>
<Accordion title="Invalid API Key">
**Error**: `{"error": true, "msg": "Invalid API key"}`

**Common Causes**:
- API key is incorrect or mistyped
- API key has been regenerated or revoked
- Using the wrong environment (dev vs. prod) API key

**Solutions**:
```bash
# Test your API key
curl -X GET https://api.hitl.sh/v1/loops \
  -H "Authorization: Bearer your_api_key_here" \
  -v

# Check response for detailed error information
```

1. Verify API key in your dashboard
2. Generate a new API key if needed
3. Ensure no extra spaces or characters
4. Check environment variables are loaded correctly
</Accordion>

<Accordion title="Wrong Header Format">
**Error**: `{"error": true, "msg": "Missing or invalid authorization header"}`

**Common Mistakes**:
- Using `X-API-Key` instead of `Authorization: Bearer`
- Missing the header entirely
- Incorrect header name capitalization
- Missing "Bearer " prefix

**Correct Format**:
```python
headers = {
    "Authorization": "Bearer your_api_key_here",  # Correct
    "Content-Type": "application/json"
}

# Wrong formats:
# "Authorization": "Bearer your_api_key_here"             # Wrong header name
# "authorization": "Bearer your_api_key_here"  # Wrong case
# "Authorization": "your_api_key_here"         # Missing Bearer prefix
```
</Accordion>

<Accordion title="API Key Permissions">
**Error**: `{"error": true, "msg": "Access denied to this resource"}`

**Cause**: API key doesn't have access to the requested resource (loops, requests created by other users)

**Solution**: 
- Verify you're accessing resources created by your account
- Check if API key has been restricted to specific operations
- Ensure you're using the correct API key for the environment
</Accordion>
</AccordionGroup>

### Rate Limiting

<AccordionGroup>
<Accordion title="429 Too Many Requests">
**Error**: `{"error": true, "msg": "API rate limit exceeded", "usage_count": 100, "usage_limit": 100, "remaining": 0}`

**Current Limits**:
- Free tier: 100 requests per hour
- Pro tier: 1,000 requests per hour  
- Enterprise: Custom limits

**Solutions**:
```python
import time
import random
from requests.exceptions import HTTPError

def api_call_with_backoff(func, max_retries=3):
    """Make API call with exponential backoff for rate limiting"""
    for attempt in range(max_retries):
        try:
            return func()
        except HTTPError as e:
            if e.response.status_code == 429:
                # Extract retry-after header if available
                retry_after = e.response.headers.get('Retry-After', '60')
                wait_time = int(retry_after) + random.uniform(1, 5)
                
                print(f"Rate limited. Waiting {wait_time:.1f} seconds...")
                time.sleep(wait_time)
                continue
            raise
    raise Exception("Max retries exceeded")

# Usage
def make_request():
    return requests.post(url, headers=headers, json=data)

result = api_call_with_backoff(make_request)
```

**Best Practices**:
1. Cache loop IDs and reuse them
2. Batch operations when possible
3. Use webhooks instead of frequent polling
4. Implement exponential backoff
5. Monitor your usage in the dashboard
</Accordion>
</AccordionGroup>

<Accordion title="Webhook Signature Verification Failing">
**Error**: Webhook receives requests but signature verification fails

**Common Issues**:
- Using wrong webhook secret
- Signature calculation incorrect
- Raw payload not used for signature verification
- Wrong header name

**Correct Implementation**:
```python
import hmac
import hashlib

def verify_webhook_signature(payload, signature, secret):
    """Verify HITL webhook signature"""
    
    # Remove sha256= prefix if present
    if signature.startswith('sha256='):
        signature = signature[7:]
    
    # Calculate expected signature using raw payload
    expected_signature = hmac.new(
        secret.encode('utf-8'),
        payload.encode('utf-8') if isinstance(payload, str) else payload,
        hashlib.sha256
    ).hexdigest()
    
    # Use secure comparison
    return hmac.compare_digest(expected_signature, signature)

# Flask example
@app.route('/webhook/hitl', methods=['POST'])
def handle_webhook():
    # Get raw payload - don't parse JSON first!
    payload = request.get_data(as_text=True)
    signature = request.headers.get('X-HITL-Signature-256')
    
    if not verify_webhook_signature(payload, signature, WEBHOOK_SECRET):
        return {'error': 'Invalid signature'}, 401
    
    # Now parse the JSON
    data = json.loads(payload)
    # ... process webhook
```

# Cache loop IDs to avoid repeated lookups
import functools
import time

@functools.lru_cache(maxsize=100)
def get_loop_id_by_name(loop_name):

def handle_response_data_safely(request_data):
    """Safely handle different response data formats"""
    response_type = request_data["response_type"]
    response_data = request_data["response_data"]
    
    if response_type == "single_select":
        # Should be string
        assert isinstance(response_data, str)
        return response_data
    
    elif response_type == "multi_select":  
        # Should be array of strings
        assert isinstance(response_data, list)
        return [str(item) for item in response_data]
    
    elif response_type == "rating":
        # Should be number
        assert isinstance(response_data, (int, float))
        return float(response_data)

        # Should be string
        assert isinstance(response_data, str)
        return response_data.strip()
    
    elif response_type == "boolean":
        # Should be boolean
        assert isinstance(response_data, bool)
        return response_data
        
    elif response_type == "number":
        # Should be number
        assert isinstance(response_data, (int, float))
        return response_data

import sys
import requests
import json
from datetime import datetime

def collect_debug_info():
    """Collect information for support requests"""
    debug_info = {
        "timestamp": datetime.now().isoformat(),
        "python_version": sys.version,
        "requests_version": requests.__version__,
        "platform": sys.platform,
    }
    
    # Test basic connectivity
    try:
        response = requests.get("https://api.hitl.sh/v1/health", timeout=10)
        debug_info["api_status"] = response.status_code
        debug_info["api_response_time_ms"] = response.elapsed.total_seconds() * 1000
    except Exception as e:
        debug_info["api_connection_error"] = str(e)
    
    # Test authentication (don't include actual API key in debug info)
    try:
        response = requests.get(
            "https://api.hitl.sh/v1/loops",
            headers={"Authorization": "Bearer your_api_key_here"},
            timeout=10
        )
        debug_info["auth_status"] = response.status_code
        if response.status_code != 200:
            debug_info["auth_error"] = response.text
    except Exception as e:
        debug_info["auth_connection_error"] = str(e)
    
    return debug_info

# Run and include output in support requests
print(json.dumps(collect_debug_info(), indent=2))
