# Home Screen

The HITL.sh mobile app home screen is your central hub for managing review requests on the go. Designed for efficiency and ease of use, it provides quick access to all the information you need to make informed decisions.

## Home Screen Overview

<Frame>
<img src="/images/requests_list.PNG" alt="HITL.sh mobile app home screen showing pending requests and navigation" />
</Frame>

The home screen is organized into several key areas:

- **Header Section**: Quick stats and navigation menu
- **Request Queue**: List of pending requests requiring your review
- **Quick Actions**: Common tasks and shortcuts
- **Status Indicators**: Visual cues for request priority and status

## Request Queue

### Pending Requests List

The main section displays all requests waiting for your review:

<Card title="Request Cards" icon="file-text">
Each request card shows:
- **Content Preview**: First few words or image thumbnail
- **Request Type**: Text, image, video, or document
- **Priority Level**: Visual indicator (low, normal, high, urgent)
- **Time Remaining**: Countdown to response deadline
- **AI Confidence**: How confident the AI was in its analysis
</Card>

### Request Prioritization

Requests are automatically sorted by:

<Steps>
<Step title="Priority Level">
Urgent requests appear at the top of the queue.
</Step>

<Step title="Response Time">
Requests approaching their deadline are highlighted.
</Step>

<Step title="Your Expertise">
Requests matching your skills are prioritized.
</Step>

<Step title="Queue Position">
First-in, first-out ordering for equal priority requests.
</Step>
</Steps>

### Request Status Indicators

Visual cues help you quickly understand request status:

<CardGroup cols={2}>
<Card title="Priority Colors" icon="palette">
- **Green**: Low priority, normal response time
- **Yellow**: Normal priority, approaching deadline
- **Orange**: High priority, requires attention
- **Red**: Urgent priority, immediate action needed
</Card>

<Card title="Status Icons" icon="info">
- **Clock**: Request is waiting for review
- **Alert**: Request is approaching deadline
- **Star**: Request matches your expertise
- **Flag**: Request has been flagged for special attention
</Card>
</CardGroup>

## Troubleshooting

### Common Issues

<AccordionGroup>
<Accordion title="App Not Loading">
- Check internet connection
- Restart the app
- Update to latest version
- Clear app cache
</Accordion>

<Accordion title="Notifications Not Working">
- Check notification permissions
- Verify notification settings
- Restart device
- Reinstall app if needed
</Accordion>

<Accordion title="Request Not Updating">
- Pull to refresh the request list
- Check internet connection
- Log out and back in
- Contact support if persistent
</Accordion>
</AccordionGroup>
